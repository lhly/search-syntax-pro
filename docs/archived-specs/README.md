# 已归档的需求规范文档

此目录包含已完成开发的历史需求规范文档，用于参考和审计目的。

## 📂 目录结构

### 已完成的功能模块

1. **comprehensive-optimization/** - 综合优化
   - 项目整体性能和代码质量优化

2. **yandex-duckduckgo-floating-button/** - Yandex & DuckDuckGo 悬浮按钮功能
   - 为 Yandex 和 DuckDuckGo 搜索引擎添加悬浮按钮支持
   - 需求规范、实现和验证文档

3. **search-engine-adapters-expansion/** - 搜索引擎适配器扩展
   - 新增多个搜索引擎适配器
   - 包括 Yahoo、Twitter、360搜索、Qwant、Sogou、Naver、Startpage、Ecosia

4. **improved-floating-panel/** - 改进的悬浮面板
   - 悬浮面板UI/UX改进
   - 性能优化和用户体验提升

5. **v1-7-0-feature-expansion/** - v1.7.0 功能扩展
   - v1.7.0 版本的功能规划和实现

6. **bing-search-input-enhancement/** - Bing 搜索输入增强
   - Bing 搜索引擎的输入框增强功能

7. **需求文档实现/** - 需求文档实现流程
   - 产品需求文档 (PRD)
   - 系统架构设计
   - Sprint 计划
   - 开发审查

## 📝 文档类型

每个功能模块通常包含以下类型的文档：

- **00-repository-context.md** / **00-repo-scan.md** - 项目上下文和代码库分析
- **requirements-spec.md** / **01-product-requirements.md** - 功能需求规范
- **requirements-confirm.md** - 需求确认
- **02-system-architecture.md** - 系统架构设计
- **03-sprint-plan.md** - 开发计划
- **04-dev-reviewed.md** - 开发审查

## 🔍 使用说明

### 查阅历史需求
当需要了解某个功能的设计背景和实现细节时，可以参考相应目录下的文档。

### 新功能参考
开发新功能时，可以参考已完成功能的文档结构和规范格式。

### 审计追溯
需要审计功能实现和需求对齐情况时，可以查看完整的需求到实现的文档链。

## ⚠️ 注意事项

- 这些文档为历史归档，反映的是开发时的状态
- 实际实现可能与原始规范有所差异（基于开发过程中的调整）
- 最新的实现细节请参考源代码和最新文档
- 不要修改已归档的文档，保持历史记录的完整性
